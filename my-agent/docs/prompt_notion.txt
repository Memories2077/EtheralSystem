Đây là tài liệu:
Notion:
Notion API Pages Endpoint
Base Information
API Endpoint: https://api.notion.com/v1/pages
Integration token: <my_token>
Obligatory Headers for All Requests:
•	Authorization: Bearer <token>
•	Notion-Version: 2022-06-28 (or latest)
•	Content-Type: application/json
POST - Create Page
Request Pattern:
POST /v1/pages HTTP/1.1
Host: api.notion.com
Authorization: Bearer YOUR_INTEGRATION_TOKEN
Notion-Version: 2022-06-28
Content-Type: application/json
Request Body:
{
  "parent": { "database_id": "your_database_id" },
  "properties": {
    "Name": {
      "title": [{ "text": { "content": "Page Title" } }]
    }
  }
}
Success Response (201):
{
  "object": "page",
  "id": "b55c9c91-384d-452b-81db-d1ef79372b75",
  "created_time": "2021-05-21T19:17:00.000Z",
  "last_edited_time": "2021-05-21T19:17:00.000Z",
  "properties": {
    "Name": {
      "title": [
        {
          "text": {
            "content": "Page Title"
          }
        }
      ]
    }
  }
}

PATCH - Update Page
Request Pattern:
PATCH /v1/pages/{page_id} HTTP/1.1
Host: api.notion.com
Authorization: Bearer YOUR_INTEGRATION_TOKEN
Notion-Version: 2022-06-28
Content-Type: application/json
URL Parameters:
•	page_id (required): The ID of the page to update
Request Body Examples:
Update Properties:
{
  "properties": {
    "Name": {
      "title": [{ "text": { "content": "Updated Page Title" } }]
    },
    "Status": {
      "select": { "name": "In Progress" }
    }
  }
}
Archive/Unarchive Page:
{
  "archived": true
}
Update Icon:
{
  "icon": {
    "type": "emoji",
    "emoji": "🎯"
  }
}
Update Cover:
{
  "cover": {
    "type": "external",
    "external": {
      "url": "https://example.com/image.jpg"
    }
  }
}
Success Response (200):
{
  "object": "page",
  "id": "b55c9c91-384d-452b-81db-d1ef79372b75",
  "created_time": "2021-05-21T19:17:00.000Z",
  "last_edited_time": "2021-05-21T19:25:00.000Z",
  "properties": {
    "Name": {
      "title": [
        {
          "text": {
            "content": "Updated Page Title"
          }
        }
      ]
    }
  }
}
Error Responses
Validation Error (400):
{
  "object": "error",
  "status": 400,
  "code": "validation_error",
  "message": "body failed validation: required property 'parent' is missing"
}
Unauthorized (401):
{
  "object": "error",
  "status": 401,
  "code": "unauthorized",
  "message": "API token is invalid"
}
Not Found (404):
{
  "object": "error",
  "status": 404,
  "code": "object_not_found",
  "message": "Could not find page with ID: invalid-id"
}
Rate Limited (429):
{
  "object": "error",
  "status": 429,
  "code": "rate_limited",
  "message": "Rate limited"
}
Important Notes
•	Object Identification: Response always has "object" parameter to determine data types (page, database, error, etc.)
•	Status Codes: Always check HTTP status code to determine success/failure
•	Rate Limits: Notion API has rate limits - handle 429 responses appropriately
•	Property Types: Different property types (title, text, select, multi_select, date, etc.) have different structures
•	Permissions: Integration must have appropriate permissions to read/write pages and databases

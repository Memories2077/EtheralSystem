# Research API Input Document
Format-Version: 1
API-ID: thedogapi-input-doc
Title: TheDogAPI Media And Account API
Base-URL: https://api.thedogapi.com/v1
Authentication: Most endpoints require x-api-key. Public image search and image lookup can be called without a key, but an API key may improve rate limits.
Declared-Endpoint-Count: 8
Safe-Probe-Policy: Only public GET image endpoints are safe for default live probes. API-key, account, upload, favourite, and mutation endpoints are skipped unless credentials are supplied.

## Endpoint 1: Search images
Method: GET
Path: /images/search
Description: Search public dog images. Optional parameters include limit, page, order, breed_id, has_breeds, include_breeds, include_categories, and lang.
Request Example:
```http
GET /images/search?limit=1&include_breeds=true HTTP/1.1
Host: api.thedogapi.com
Accept: application/json
```
Response Example:
```json
[
  { "id": "abc123", "url": "https://cdn2.thedogapi.com/images/abc123.jpg", "width": 600, "height": 400, "breeds": [] }
]
```
Safe Probe: yes, use for search-images with limit=1 and include_breeds=true.

## Endpoint 2: Get image by id
Method: GET
Path: /images/{id}
Description: Return one public image by id.
Request Example:
```http
GET /images/abc123 HTTP/1.1
Host: api.thedogapi.com
Accept: application/json
```
Response Example:
```json
{ "id": "abc123", "url": "https://cdn2.thedogapi.com/images/abc123.jpg", "width": 600, "height": 400, "breeds": [] }
```
Safe Probe: skip by default because the benchmark does not know a stable generated image id before search.

## Endpoint 3: Get image status
Method: GET
Path: /images/{id}/status
Description: Return processing or moderation status for an image. Requires API key.
Request Example:
```http
GET /images/abc123/status HTTP/1.1
Host: api.thedogapi.com
x-api-key: your_dog_api_key
Accept: application/json
```
Response Example:
```json
{ "id": "abc123", "status": "labelled", "updated_at": "2026-01-01T00:00:00.000Z" }
```
Safe Probe: skip because API key is required.

## Endpoint 4: List breeds
Method: GET
Path: /breeds
Description: Return dog breed resources. Requires API key in this benchmark fixture.
Request Example:
```http
GET /breeds?limit=10&page=0 HTTP/1.1
Host: api.thedogapi.com
x-api-key: your_dog_api_key
Accept: application/json
```
Response Example:
```json
[
  { "id": "1", "name": "Affenpinscher", "temperament": "Stubborn, Curious, Playful", "life_span": "10 - 12 years" }
]
```
Safe Probe: skip because API key is required.

## Endpoint 6: List favourites
Method: GET
Path: /favourites
Description: Return favourite images for an account. Requires API key or bearer token.
Request Example:
```http
GET /favourites?limit=10&page=0 HTTP/1.1
Host: api.thedogapi.com
x-api-key: your_dog_api_key
Accept: application/json
```
Response Example:
```json
[
  { "id": 1, "image_id": "abc123", "sub_id": "benchmark", "created_at": "2026-01-01T00:00:00.000Z" }
]
```
Safe Probe: skip because account credentials are required.

## Endpoint 7: Create favourite
Method: POST
Path: /favourites
Description: Create a favourite image for an account. Requires API key or bearer token.
Request Example:
```http
POST /favourites HTTP/1.1
Host: api.thedogapi.com
x-api-key: your_dog_api_key
Content-Type: application/json

{ "image_id": "abc123", "sub_id": "benchmark" }
```
Response Example:
```json
{ "id": 1, "message": "SUCCESS" }
```
Safe Probe: skip because account credentials are required and the call mutates account state.

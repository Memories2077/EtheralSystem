# Research API Input Document
Format-Version: 1
API-ID: pokeapi-input-doc
Title: PokeAPI Read-Only Resource API
Base-URL: https://pokeapi.co/api/v2
Authentication: none
Declared-Endpoint-Count: 8
Safe-Probe-Policy: Public read-only API. All documented endpoints are GET requests and safe for low-volume live probes. Use small list limits and stable ids or names.

## Endpoint 1: List Pokemon
Method: GET
Path: /pokemon
Description: Return a paginated list of Pokemon resources. Optional query parameters include limit and offset.
Request Example:
```http
GET /pokemon?limit=5&offset=0 HTTP/1.1
Host: pokeapi.co
Accept: application/json
```
Response Example:
```json
{
  "count": 1328,
  "next": "https://pokeapi.co/api/v2/pokemon?offset=5&limit=5",
  "previous": null,
  "results": [
    { "name": "bulbasaur", "url": "https://pokeapi.co/api/v2/pokemon/1/" }
  ]
}
```
Safe Probe: yes, use limit=5 and offset=0.

## Endpoint 2: Get Pokemon by id or name
Method: GET
Path: /pokemon/{id_or_name}
Description: Return one Pokemon resource by numeric id or resource name.
Request Example:
```http
GET /pokemon/pikachu HTTP/1.1
Host: pokeapi.co
Accept: application/json
```
Response Example:
```json
{
  "id": 25,
  "name": "pikachu",
  "base_experience": 112,
  "height": 4,
  "weight": 60,
  "abilities": [
    { "ability": { "name": "static", "url": "https://pokeapi.co/api/v2/ability/9/" }, "is_hidden": false, "slot": 1 }
  ],
  "types": [
    { "slot": 1, "type": { "name": "electric", "url": "https://pokeapi.co/api/v2/type/13/" } }
  ]
}
```
Safe Probe: yes, use id_or_name=pikachu.

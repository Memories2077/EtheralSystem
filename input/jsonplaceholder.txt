# Research API Input Document
Format-Version: 1
API-ID: jsonplaceholder-input-doc
Title: JSONPlaceholder CRUD API
Base-URL: https://jsonplaceholder.typicode.com
Authentication: none
Declared-Endpoint-Count: 8
Safe-Probe-Policy: Public fake REST API. POST, PUT, PATCH, and DELETE return simulated responses and do not persist data. Safe benchmark probes may call GET endpoints and fake POST /posts.

## Endpoint 1: List posts
Method: GET
Path: /posts
Description: Return a list of post resources. Optional query parameter userId filters posts by user.
Request Example:
```http
GET /posts?userId=1 HTTP/1.1
Host: jsonplaceholder.typicode.com
Accept: application/json
```
Response Example:
```json
[
  { "userId": 1, "id": 1, "title": "sample title", "body": "sample body" }
]
```
Safe Probe: yes, use for list-posts with userId=1 or no arguments.

## Endpoint 2: Get post by id
Method: GET
Path: /posts/{id}
Description: Return one post resource by numeric id.
Request Example:
```http
GET /posts/1 HTTP/1.1
Host: jsonplaceholder.typicode.com
Accept: application/json
```
Response Example:
```json
{ "userId": 1, "id": 1, "title": "sample title", "body": "sample body" }
```
Safe Probe: yes, use id=1.

## Endpoint 3: Create post
Method: POST
Path: /posts
Description: Create a fake post. The service returns a simulated id and does not persist the post.
Request Example:
```http
POST /posts HTTP/1.1
Host: jsonplaceholder.typicode.com
Content-Type: application/json

{ "title": "benchmark", "body": "tool call", "userId": 1 }
```
Response Example:
```json
{ "title": "benchmark", "body": "tool call", "userId": 1, "id": 101 }
```
Safe Probe: yes, use for create-post.

## Endpoint 4: Replace post
Method: PUT
Path: /posts/{id}
Description: Replace an entire fake post resource. The service returns a simulated updated object.
Request Example:
```http
PUT /posts/1 HTTP/1.1
Host: jsonplaceholder.typicode.com
Content-Type: application/json

{ "id": 1, "title": "updated", "body": "updated body", "userId": 1 }
```
Response Example:
```json
{ "id": 1, "title": "updated", "body": "updated body", "userId": 1 }
```
Safe Probe: skip by default because mutation probes are not needed for success-rate calculation.

## Endpoint 7: List comments for a post
Method: GET
Path: /posts/{id}/comments
Description: Return comments for a post id. Equivalent to GET /comments?postId={id}.
Request Example:
```http
GET /posts/1/comments HTTP/1.1
Host: jsonplaceholder.typicode.com
Accept: application/json
```
Response Example:
```json
[
  { "postId": 1, "id": 1, "name": "sample name", "email": "user@example.com", "body": "sample comment" }
]
```
Safe Probe: yes, use for post-comments with postId=1 or id=1.
